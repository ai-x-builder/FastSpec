# REVIEW.md

Use this structure:

```markdown
# Review

## Summary

<Review summary.>

## Scope Review

<Scope findings.>

## Spec Compliance

<Spec compliance findings.>

## Code Quality

<Quality findings.>

## Risks

<Remaining risks or `None`.>

## Reviewer Decision

<pending | accepted | changes_requested | blocked | needs_product_review | needs_tech_review>
```

Use `pending` only while implementation or review is still in progress. Successful completion requires `accepted`.
